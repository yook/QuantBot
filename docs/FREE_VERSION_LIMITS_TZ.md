# ТЗ: ограничения бесплатной версии PageViewer

## 1. Цель

Добавить ограничения для бесплатной версии PageViewer, чтобы пользователь мог полноценно попробовать продукт на небольших проектах, а регулярная работа с большим объемом данных переходила в платную версию.

## 2. Ограничения Free-версии

В бесплатной версии должны действовать следующие лимиты:

- максимум 3 проекта;
- до 1 000 URL на один проект;
- значение `Макс. URL` по умолчанию равно 1 000;
- экспорт XLS до 1 000 строк;
- параллельность краулинга не больше 1 потока;
- значение `Параллельность` по умолчанию равно 1;
- интервал между запросами не меньше 1000 мс;
- значение `Интервал, мс` по умолчанию равно 1000;
- рендеринг страниц доступен только в Pro-версии.

## 3. Правила применения лимитов

### 3.1. Лимит проектов

Пользователь Free-версии может создать не более 3 проектов.

Если пользователь пытается создать 4-й проект:

- проект не создается;
- показывается уведомление;
- пользователь получает понятное объяснение причины и предложение перейти на платную версию.

Текст уведомления:

`В бесплатной версии можно создать до 3 проектов. Чтобы добавить больше проектов, перейдите на Pro.`

### 3.2. Лимит URL на проект

В одном проекте Free-версии может быть не более 1 000 URL.

Настройка `Макс. URL` должна быть доступна в интерфейсе и использовать лимит Free-версии:

- значение по умолчанию для Free-версии: 1 000;
- значение `0` запрещено, так как в текущем смысле может означать отсутствие ограничения;
- значение больше 1 000 запрещено;
- если пользователь вводит `0` или значение больше 1 000, настройка должна возвращаться к 1 000 или к последнему допустимому значению;
- пользователь должен увидеть сообщение об ограничении.

Текст уведомления при вводе `0`:

`В бесплатной версии нельзя отключить лимит URL. Максимум — 1 000 URL на проект.`

Текст уведомления при вводе значения больше 1 000:

`В бесплатной версии доступно до 1 000 URL на проект. Для большего лимита перейдите на Pro.`

Лимит должен учитывать URL, которые сохраняются в проекте в результате:

- краулинга сайта;
- ручной загрузки URL для парсинга;
- очереди и уже проверенных URL, если они сохраняются как данные проекта.

Если лимит достигнут:

- новые URL не добавляются сверх лимита;
- текущий процесс должен корректно остановиться или пропустить добавление новых URL;
- пользователь должен увидеть сообщение о достижении лимита.

Текст уведомления:

`В бесплатной версии доступно до 1 000 URL на проект. Для проверки большего сайта перейдите на Pro.`

Если при импорте списка URL пользователь загружает больше доступного остатка:

- сохранить только URL в пределах лимита;
- остальные URL пропустить;
- показать количество сохраненных и пропущенных URL.

Пример уведомления:

`Сохранено 320 URL. Пропущено 180 URL из-за лимита бесплатной версии.`

### 3.3. Лимит экспорта XLS

В Free-версии экспорт XLS ограничен 1 000 строками.

Если в текущей таблице или отфильтрованном наборе больше 1 000 строк:

- экспортировать только первые 1 000 строк;
- показать уведомление до или после экспорта;
- в уведомлении указать, что полный экспорт доступен в Pro.

Текст уведомления:

`В бесплатной версии экспорт ограничен 1 000 строками. Для полного экспорта перейдите на Pro.`

### 3.4. Ограничение параллельности

В Free-версии значение параллельности краулинга не может быть больше 1.

Требования:

- значение по умолчанию для Free-версии: 1;
- в настройках краулера максимальное значение поля параллельности должно быть равно 1;
- значение `0` запрещено;
- значение больше 1 запрещено;
- если пользователь вводит `0` или значение больше 1, настройка должна возвращаться к 1 или к последнему допустимому значению;
- пользователь должен увидеть сообщение об ограничении;
- если в проекте уже сохранено значение больше 1, при запуске Free-версии оно должно быть автоматически приведено к 1;
- при запуске краулинга backend/worker также должен проверять значение и не позволять обойти лимит через сохраненную конфигурацию.

Текст уведомления при вводе `0`:

`В бесплатной версии параллельность не может быть меньше 1 потока.`

Текст уведомления при вводе значения больше 1:

`В бесплатной версии доступен 1 поток. Больше потоков доступно в Pro.`

Текст подсказки рядом с настройкой:

`В бесплатной версии доступен 1 поток. Больше потоков доступно в Pro.`

### 3.5. Ограничение интервала

В Free-версии интервал между запросами не может быть меньше 1000 мс.

Требования:

- значение по умолчанию для Free-версии: 1000 мс;
- в настройках краулера минимальное значение поля интервала должно быть 1000 мс;
- значение `0` запрещено;
- значение меньше 1000 запрещено;
- если пользователь вводит `0` или значение меньше 1000, настройка должна возвращаться к 1000 мс или к последнему допустимому значению;
- пользователь должен увидеть сообщение об ограничении;
- если в проекте уже сохранено значение меньше 1000 мс, при запуске Free-версии оно должно быть автоматически приведено к 1000 мс;
- при запуске краулинга backend/worker также должен проверять значение и не позволять обойти лимит через сохраненную конфигурацию.

Текст уведомления при вводе `0` или значения меньше 1000:

`В бесплатной версии минимальный интервал между запросами — 1000 мс.`

Текст подсказки рядом с настройкой:

`В бесплатной версии минимальный интервал между запросами — 1000 мс. Меньший интервал доступен в Pro.`

### 3.6. Ограничение рендеринга страниц

В Free-версии рендеринг страниц недоступен.

Требования:

- настройка рендеринга должна быть выключена по умолчанию;
- Free-пользователь не может включить Chromium rendering;
- если в проекте уже сохранено `renderEnabled = true`, при запуске Free-версии значение должно быть автоматически приведено к `false`;
- при запуске краулинга backend/worker также должен проверять значение и не позволять обойти лимит через сохраненную конфигурацию.

Текст уведомления при попытке включить рендеринг:

`Рендеринг страниц доступен только в Pro-версии.`

## 4. Поведение интерфейса

### 4.1. Настройки краулера

В настройках краулера для Free-версии:

- поле `Макс. URL` отображается со значением по умолчанию 1 000;
- поле `Макс. URL` не позволяет сохранить `0` или значение больше 1 000;
- при попытке ввести недопустимое значение пользователь видит уведомление об ограничении Free-версии;
- поле `Параллельность` отображается со значением по умолчанию 1;
- поле `Параллельность` не позволяет сохранить `0` или значение больше 1;
- при попытке ввести недопустимую параллельность пользователь видит уведомление об ограничении Free-версии;
- поле `Интервал, мс` отображается со значением по умолчанию 1000;
- поле `Интервал, мс` не позволяет сохранить `0` или значение меньше 1000;
- при попытке ввести недопустимый интервал пользователь видит уведомление об ограничении Free-версии;
- рядом с ограниченными настройками должна быть подсказка о лимите Free-версии.

### 4.2. Создание проекта

Если лимит проектов достигнут:

- кнопка создания проекта может оставаться активной, но при попытке создания должен появляться диалог/уведомление;
- предпочтительно показать понятный upgrade CTA: `Перейти на Pro`.

### 4.3. Таблица и экспорт

При экспорте:

- если строк не больше 1 000, экспорт работает как обычно;
- если строк больше 1 000, экспорт ограничивается первыми 1 000 строками;
- пользователь видит уведомление об ограничении.

### 4.4. Краулинг и загрузка URL

При достижении 1 000 URL:

- кнопки и процессы не должны ломаться;
- пользователь должен понимать, почему новые URL не добавляются;
- если процесс краулинга был остановлен лимитом, прогресс не должен зависать в состоянии выполнения.

### 4.5. Рендеринг страниц

В настройках рендеринга для Free-версии:

- переключатель рендеринга отображается выключенным;
- включение переключателя блокируется или сразу возвращается в выключенное состояние;
- пользователь видит уведомление, что рендеринг доступен только в Pro;
- поля таймаута и параллельности рендера остаются недоступными.

## 5. Технические требования

### 5.1. Единый источник лимитов

Лимиты Free-версии должны быть описаны в одном месте, чтобы их можно было переиспользовать в UI, IPC и worker-логике.

Рекомендуемая структура:

```ts
export const FREE_PLAN_LIMITS = {
  projects: 3,
  urlsPerProject: 1000,
  defaultMaxUrls: 1000,
  exportRows: 1000,
  crawlerConcurrency: 1,
  defaultCrawlerConcurrency: 1,
  minCrawlerIntervalMs: 1000,
  defaultCrawlerIntervalMs: 1000,
  renderingEnabled: false,
};
```

### 5.2. Проверка на уровне UI и backend

Ограничения должны проверяться не только в интерфейсе, но и в IPC/backend-слое.

Причина: пользовательские настройки могут быть сохранены раньше, изменены вручную или переданы в обход UI.

Проверять нужно:

- создание проекта;
- добавление URL;
- запуск краулинга;
- запуск парсинга, если он добавляет URL в проект;
- экспорт XLS;
- сохранение настроек краулера;
- сохранение настроек рендеринга;

### 5.3. Сохранение настроек

Если Free-пользователь пытается сохранить настройки вне лимитов:

- `maxUrls` приводится к 1 000, если значение равно 0 или больше 1 000;
- `maxConcurrency` приводится к 1, если значение равно 0 или больше 1;
- `interval` приводится к 1000 мс, если значение равно 0 или меньше 1000;
- `renderEnabled` приводится к `false`, если пользователь Free-версии пытается включить рендеринг;
- пользователь получает понятную подсказку, если изменение произошло через интерфейс.

## 7. Критерии приемки

- Free-пользователь не может создать больше 3 проектов.
- В одном проекте Free-версии не сохраняется больше 1 000 URL.
- В Free-версии значение `Макс. URL` по умолчанию равно 1 000.
- Пользователь Free-версии не может сохранить `Макс. URL = 0`.
- Пользователь Free-версии не может сохранить `Макс. URL` больше 1 000.
- При попытке ввести `0` или значение больше 1 000 пользователь видит сообщение об ограничении.
- При загрузке URL сверх лимита лишние URL пропускаются, а пользователь видит уведомление.
- XLS-экспорт Free-версии не содержит больше 1 000 строк.
- В Free-версии значение `Параллельность` по умолчанию равно 1.
- Пользователь Free-версии не может сохранить `Параллельность = 0`.
- Пользователь Free-версии не может сохранить `Параллельность` больше 1.
- При попытке ввести недопустимую параллельность пользователь видит сообщение об ограничении.
- Параллельность краулинга в Free-версии не может быть больше 1.
- В Free-версии значение `Интервал, мс` по умолчанию равно 1000.
- Пользователь Free-версии не может сохранить `Интервал, мс = 0`.
- Пользователь Free-версии не может сохранить `Интервал, мс` меньше 1000.
- При попытке ввести недопустимый интервал пользователь видит сообщение об ограничении.
- Интервал краулинга в Free-версии не может быть меньше 1000 мс.
- Рендеринг страниц в Free-версии выключен по умолчанию.
- Free-пользователь не может включить рендеринг страниц.
- Если в сохраненной конфигурации Free-проекта рендеринг включен, приложение приводит `renderEnabled` к `false`.
- Лимиты нельзя обойти через уже сохраненные настройки проекта.
- При достижении лимитов процессы завершаются корректно, без зависшего прогресса.
- Все уведомления написаны понятным языком и объясняют, что полный функционал доступен в Pro.

## 8. Что нужно уточнить перед разработкой

- Как приложение определяет тариф пользователя: локальная лицензия, аккаунт, флаг сборки или remote config.
- Нужно ли полностью запрещать экспорт больше 1 000 строк или экспортировать первые 1 000 строк.
- Должен ли лимит 1 000 URL учитывать только проверенные URL или все URL проекта, включая очередь, запрещенные и URL, загруженные для парсинга.
- Нужна ли отдельная визуальная плашка `Free`/`Pro` в интерфейсе.
- Какой URL должен открываться по CTA `Перейти на Pro`.
